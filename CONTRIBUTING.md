## Français

### Qui peut contribuer ?

Tout le monde. Vous n’êtes pas obligé(e) d’être Burkinabè ni d’être développeur professionnel. Si vous avez un projet open source utile pour le Burkina Faso ou l’Afrique subsaharienne, il a sa place ici.

### Critères d’inclusion

Un projet doit respecter **tous** les critères suivants pour être accepté :

1. **Open source** : le code source est publiquement accessible (GitHub, GitLab, Codeberg, etc.) sous une licence reconnue (MIT, Apache 2.0, GPL, MPL, CC0…).
2. **Pertinence** : le projet est créé par une personne basée au Burkina Faso, **ou** il est explicitement conçu pour répondre à un besoin burkinabè ou africain.
3. **Actif ou archivé consciemment** : le projet n’est pas abandonné sans documentation. Les projets archivés doivent être clairement signalés `[archivé]`.
4. **Documenté** : le dépôt possède au minimum un README lisible.

### Comment soumettre un projet ?

**Option 1 : Pull Request (recommandée)**

1. Forkez ce dépôt.
2. Modifiez le fichier `README.md` : ajoutez une ligne dans la bonne catégorie en respectant le format du tableau.
3. Soumettez une Pull Request avec le titre : `Add: [Nom du projet]`.
4. Dans la description de la PR, expliquez en deux ou trois phrases pourquoi ce projet mérite d’être listé.

**Option 2 : Issue**

Ouvrez une issue en utilisant le modèle [Proposer un outil](.github/ISSUE_TEMPLATE/add_tool.md).

### Format du tableau

```markdown
| [Nom du projet](lien) | Description courte en français. Short description in English. | Langage principal | [Auteur](lien profil) |
```

### Ce qui sera refusé

- Projets propriétaires ou à code source fermé.
- Projets sans rapport avec le Burkina Faso ou l’Afrique subsaharienne.
- Liens brisés ou dépôts vides.
- Autopromotion commerciale déguisée.

## English

### Who can contribute?

Anyone. You don't need to be Burkinabè or a professional developer. If you have an open source project useful for Burkina Faso or sub-Saharan Africa, it belongs here.

### Inclusion criteria

A project must meet **all** of the following criteria to be accepted:

1. **Open source**: source code is publicly accessible (GitHub, GitLab, Codeberg, etc.) under a recognized license (MIT, Apache 2.0, GPL, MPL, CC0…).
2. **Relevance**: the project is created by someone based in Burkina Faso, **or** it is explicitly designed to address a Burkinabè or African need.
3. **Active or consciously archived**: the project is not silently abandoned. Archived projects must be clearly marked `[archived]`.
4. **Documented**: the repository has at least a readable README.

### How to submit a project?

**Option 1 : Pull Request (recommended)**

1. Fork this repository.
2. Edit the `README.md` file: add a row in the correct category following the table format.
3. Submit a Pull Request with the title: `Add: [Project Name]`.
4. In the PR description, explain in two or three sentences why this project deserves to be listed.

**Option 2 : Issue**

Open an issue using the [Propose a tool](.github/ISSUE_TEMPLATE/add_tool.md) template.

### Table format

```markdown
| [Project name](link) | Short description in English. Description courte en français. | Main language | [Author](profile link) |
```

### What will be rejected

- Proprietary or closed-source projects.
- Projects unrelated to Burkina Faso or sub-Saharan Africa.
- Broken links or empty repositories.
- Disguised commercial self-promotion.
